# Summary

The `mine-bean` skill runs on a 5-minute cron but cannot execute its core function. Three independent blockers must all be addressed before any on-chain deploy or claim happens:

## Blockers

1. **Secrets not exported.** `.github/workflows/aeon.yml` does not pass `BASE_RPC_URL`, `AGENT_ADDRESS`, or `AGENT_PRIVATE_KEY` into the `Run` step's `env:` block. The skill's scripts hard-fail with `BASE_RPC_URL is required`.
2. **`cast` not installed.** The runner has no Foundry toolchain. Every script (`status.sh`, `deploy.sh`, `claim.sh`) shells out to `cast call` / `cast send` / `cast balance`.
3. **Tool allow-list too narrow.** The workflow restricts Claude to a fixed bash allow-list (`curl`, `gh`, `git`, `jq`, `./notify`, `mkdir`, `ls`, `cat`, `chmod`, `date`, `echo`, `node`, `npm`, `npx`, `head`, `tail`, `wc`, `sort`, `grep`). It excludes `Bash(cast:*)` and `Bash(./scripts/*)`, so even with cast installed Claude couldn't invoke the skill's scripts.

## Fix plan

A single PR should:
- Add a Foundry install step to the workflow (e.g. `foundry-rs/foundry-toolchain@v1`).
- Extend the `Run` step's `env:` block with `BASE_RPC_URL: ${{ secrets.BASE_RPC_URL }}`, `AGENT_ADDRESS: ${{ secrets.AGENT_ADDRESS }}`, `AGENT_PRIVATE_KEY: ${{ secrets.AGENT_PRIVATE_KEY }}` plus the optional config knobs (`DRY_RUN`, `DEPLOY_BLOCKS`, etc.) as repo `vars`.
- Add `Bash(cast:*)` and `Bash(./scripts/*)` to the `ALLOWED` string in the `Run` step.
- Configure the three GitHub Actions secrets and fund the EOA with 0.01–0.05 ETH on Base.
- First production run should use `DRY_RUN=true` until the deploy path is validated.

## Notes

- Until this is fixed, each cron fire will land here, log SKIPPED, and exit cleanly. No funds at risk.
- The skill itself (SKILL.md + scripts) looks production-ready; only the host wiring is missing.

## Resolution

Fix landed in commit `8c24aba`: Foundry toolchain install step added, `BASE_RPC_URL` / `AGENT_ADDRESS` / `AGENT_PRIVATE_KEY` exported into the `Run` step `env:` block, and `Bash(cast:*)` / `Bash(bash:*)` / `Bash(sed:*)` / `Bash(awk:*)` / `Bash(printf:*)` added to the `ALLOWED` tool string.
